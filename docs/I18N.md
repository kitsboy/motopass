# Internationalization Reference

**BUILD:** 2026.07.14-28 · **Last updated:** 2026-07-14

MotoPass uses a two-layer i18n system:

1. **Nav & shell** — `src/i18n/translations.ts` (10 locales: en, es, fr, de, pt, zh, ar, hi, sw)
2. **Page copy** — `src/i18n/pageKeys.ts` (`PAGE_KEYS` + `pageKeysEn` + locale overrides)

## Page keys (BTC Map — BUILD 26)

Recent additions in `pageKeys.ts`:

| Key | English default |
|-----|-----------------|
| `btcmap.densityBadge` | `{count} BTC venues` |
| `btcmap.densityTooltip` | `{count} nearby merchants · density score {score}/100` |
| `btcmap.save` / `btcmap.unsave` | Save / remove saved merchant |
| `btcmap.signInSave` | Sign in to save favorites |
| `btcmap.offlineCache` | Offline snapshot |
| `btcmap.reportTitle` | Report a Bitcoin venue |
| `btcmap.reportWeb` | Add on btcmap.org |
| `btcmap.reportCli` | btcmap-cli |

Use `formatT(t, key, { count, score })` for interpolated strings.

## RTL

Arabic (`ar`) sets `document.documentElement.dir = 'rtl'` via `I18nContext`. E2e test covers this.

## Shared Give A Bit translations

Master translations may also live at `~/projects/giveabit/src/i18n/` for cross-project terms. MotoPass-specific strings use the `pageKeys` system above.

## Adding a new locale override

1. Add keys to `PAGE_KEYS` array in `pageKeys.ts`
2. Add English default in `pageKeysEn`
3. Add partial override in `pageKeysEs`, `pageKeysFr`, etc.
4. Run `npm test` — page key coverage is type-checked via `PageKey` union
